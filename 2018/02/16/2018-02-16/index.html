  • Spacemacs: paste into minibuffer
  • Is there a way to paste something into the minibuffer?
  • For example if I want to eval-expression and paste soemthing in there neither C-v or (obviously p) will work? There has to be a way.
  • Answer: C-y does work, but you can also paste into the minibuffer from the default register with, e.g., M0m r e “ or M-m r e return – or by searching the contest of all registers: for example M-m r e test copied a while ago.
  • M-m is the default dotspacemacs-emacs-leader-key : it’s similar to SPC (which is hte default dotspacemacs-leader-ky), but the former is available where SPC would just insert a literal space character (like int the minibuffer). Personally, I’ve just set dotspacemacs-emacs-leadr-key to a very easily accessible key chord and used it everywhere instead of SPC.
    Currently, C-c doesn’t access evil’s registers from within the minibuffer (though it does work eslewhere).
  • EDIT: When in ex-mode(:), you can indeed use C-r to paste from register into the minibuffer. Additionall,hy, you can switch to evil’s search module ( evvil-search-module ) in order to enable the same when searching with /. The evil search module also enables eil-delete-backward-word within the search minibuffer ( on C-w), as well as pasting the oword under the non-minibuffer’s cursor with C-r C-w.
  • EDIT: and if you’re using ivy, M-m r y ( counsel-yank-pop – this allows you to search your kill ring, which includes a lengthier history of copied text than the evil registers) also works in the minibuffer.
  • The dotspacemacs-emacs-leader-key is M-m, which means “Alt + m” in my keyboard.
  • GOPATH 环境变量用于指定这样一些目录：除$GOROOT 之外的包含GO项目源代码和二进制文件的目录。go install 和 go 工具会用到 GOPATH：作为编译后的二进制的存放目的地和import包时的搜索路径。
  • GOPATH 是一个路径列表，也就是可以同时指定多个目录。多个目录在Mac和Linux下通过”:”分割；Windows下通过”;”分割。
  • 个人建议GOPATH中别设置多个路径，甚至不显示指定GOPATH。有些地方建议在设置了GOPATH之后，将$GOPATH/bin加入PATH中，这样可以方便的运行go install好的二进制程序。然而，当存在GOPATH中存在的多个路径时，这种写法只会将最后一个路径跟上bin。 在mac或linux下可以通过这种方式解决：${GOPATH//://bin:}/bin

  • Flask的request.form和request.data有什么区别？首先使用这两个方法的前提是post或者put请求。两者的区别在于处理不同的mimetype类型的数据，返回值也不同。当minitype为application/x-www-form-urlencodeed或者multipart/form-data的时候，也就是我们所谓表单提交，访问request.form会返回一个包含解析过的表单对象的MultiDict,而request.data是空的。当flask遇到不能处理的mimetype时，请求的数据就不能被其他方式正常解析，这些方式包括request.form、request.json和request.files这几个常用的用来访问数据的属性。这时就把数据作为字符串存在request.data中。这里注意一下request.json需要application/json的mimetype类型。知道了这些处理数据的过程，那我们就可以对提交的数据进行拓展，定义一些自己专用的mimetype类型，并在Request类中定义处理专用mimetpe数据的方法，从而让我们实现更个性、与众不同的功能需求。
